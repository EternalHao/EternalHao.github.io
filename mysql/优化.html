# 1. 谈谈你理解的SQL优化

这个问题其实很大,可以说很长时间,但是你要慢慢说

在应用开发中,由于刚开始数据量小,我们更加重视功能上的实现,但是系统正式上线后,随着生产数据日渐增多,我们所写的SQL语句开始出现了超时等性能问题,就需要对SQL进行性能优化,举一个线上的实在例子,(前面的都是废话),当你发现了一个SQL比较慢的时候,首先要确定这个SQL是一直慢还是偶尔慢,确定是一直慢的时候,就开始优化

  1. 通过慢查询日志 long_query_time 确定执行效率低的SQL
  2. 通过EXPLAIN 分析有问题的SQL

这里可以大量的说EXPLAIN的各种属性,保证你很熟

  1. 通过 show profile 继续分析 通过这个你可以清楚的看到SQL执行过程中 每个线程消耗的时间
  2. 通过 trace 分析优化器如何 选择执行计划
